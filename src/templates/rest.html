LOST ReST API Calls

.timestamp does not get called as a parameter for client scripts

API CALL ARGUMENTS DESCRIPTION
lost_key request a new key
activate_user .timestamp, username add new user to LOST
suspend_user .timestamp, username remove user from LOST
list_products .timestamp, vendor, description, compartments return list of all products that meet filter
add_products .timestamp, new_products{vendor, description, alt_description, compartments} add each block of new_products to database
add_asset .timestamp, vendor, description, compartments, facility add a new asset to LOST
